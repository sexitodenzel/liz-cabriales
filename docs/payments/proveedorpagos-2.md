# proveedorpagos.md

## Contexto previo

### Métodos de pago aceptados
- Tarjeta de crédito/débito mediante pasarela de pagos
- No se acepta efectivo
- No se acepta transferencia bancaria
- No se requiere comprobante manual del cliente

### Verificación
Se requiere verificación manual por parte del admin para confirmar ciertos pagos. El flujo exacto de esta verificación está sin definir.

### Proveedor de pagos
Sin definir. Opciones en consideración:
- **MercadoPago** — recomendado: mercado mexicano, OXXO nativo, amplia documentación en español, familiar para el público objetivo
- **Stripe** — excelente documentación, soporte MXN, OXXO requiere configuración adicional
- **Openpay** — solución local BBVA, menor comunidad y documentación
- **PayPal** — descartable para este perfil de cliente

### Política de pagos
- Citas: pago 100% adelantado, sin reembolso si no cancela con 1 día de anticipación
- Cursos: pendiente definir
- Productos: pendiente definir política de devolución
